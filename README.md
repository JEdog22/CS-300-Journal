# CS 300 - Data Structures and Algorithms

## Portfolio Overview

This repository contains artifacts from my CS 300 course, including my analysis of data structure run-time and memory performance from Project One and the completed Binary Search Tree application from Project Two. These projects demonstrate my understanding of data structures, algorithms, and efficient software design while solving real-world programming problems.

---

## Reflection

### What was the problem you were solving in the projects for this course?

The primary problem I solved throughout this course was designing software to help academic advisors efficiently manage and retrieve course information for ABC University. The application needed to load course data from a CSV file, organize it using appropriate data structures, display courses in alphanumeric order, and allow users to quickly search for detailed information about individual courses. Choosing the right data structure was essential to meeting both performance and usability requirements.

### How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by first evaluating several data structures, including vectors, hash tables, and binary search trees. I analyzed their time complexity, memory usage, and suitability for the application's requirements before selecting the Binary Search Tree for the final implementation. Understanding data structures is important because they directly affect how efficiently software stores, organizes, and retrieves information. Selecting the appropriate structure can significantly improve an application's performance and scalability.

### How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest challenges was correctly reading and parsing the CSV input file while building the Binary Search Tree without introducing errors. I also encountered issues with pointer management, recursive traversal, and debugging compiler errors. I overcame these obstacles by testing each function individually, reviewing my algorithm step by step, and refining my code until it produced the expected output. Breaking the project into smaller, manageable tasks made debugging much easier.

### How has your work on this project expanded your approach to designing software and developing programs?

This project reinforced the importance of planning before writing code. Developing pseudocode, analyzing algorithm efficiency, and selecting an appropriate data structure helped me better understand how software architecture influences the final product. I now spend more time considering performance, scalability, and future maintenance before beginning implementation, rather than focusing only on making the program function correctly.

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

Working through these projects taught me the value of writing clean, organized, and modular code. I learned to separate functionality into reusable functions, use descriptive variable names, and include meaningful comments to improve readability. I also designed the program so that components such as file loading, searching, and traversal could be modified independently, making the software easier to maintain and extend with additional features in the future.
